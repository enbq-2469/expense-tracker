## Summary

Build a full-stack personal finance management web application using **Next.js 14 (App Router)**
as the unified frontend + backend runtime, **Tailwind CSS** for mobile-first responsive
styling, and **PostgreSQL** via **Prisma ORM** for persistent storage and schema migrations.

The application exposes five pages (Sign Up, Login, Home, Import, Settings) to authenticated
users. Core capabilities: JWT-based auth stored in HttpOnly cookies, CRUD transaction
management with soft-delete, per-user category management seeded from global master data
via a dedicated Prisma migration, pie-chart dashboards with four time-period filters, a monthly grouped bar chart with
year-selector for time-series income/expense comparison, and Markdown rendering in
transaction notes with DOMPurify sanitization.

## Technical Context

**Language/Version**: TypeScript 5.4, Node.js 20 LTS  
**Frontend Framework**: Next.js 14 (App Router, React Server Components)  
**Styling**: Tailwind CSS v3 — single `tailwind.config.ts` token source; no inline magic numbers  
**Markdown**: `markdown-it` v14 + `dompurify` v3 — single shared `src/lib/markdown.ts` module  
**Charts**: `recharts` v2 — pie charts (category breakdown) + grouped bar chart (monthly income vs expense, 12-month axis, year selector)  
**Auth**: JWT (access token 15 min, refresh token 7 days) stored in HttpOnly Secure cookies;
passwords hashed with `bcryptjs` (cost 12)  
**ORM & Migrations**: Prisma v5 — `prisma migrate dev` / `prisma migrate deploy`  
**Database**: PostgreSQL 16 (dev: Docker Compose; prod: managed PaaS)  
**Validation**: `zod` v3 — shared schemas used on both server Route Handlers and client forms  
**HTTP Client**: native `fetch` via a single `src/lib/api-client.ts` wrapper  
**Testing**: Vitest (unit), Playwright (e2e) — see [quickstart.md](./quickstart.md)  
**Target Platform**: Web (modern browsers: Chrome, Firefox, Safari, Edge — latest 2 major versions)  
**Project Type**: Full-stack web application (Next.js monolith)  
**Performance Goals**: Core Web Vitals green on mobile; chart update < 1 s (SC-003)  
**Constraints**: Amounts stored as integers (VND smallest unit = 1 đồng); no localStorage
for auth tokens; HTTPS enforced in production  
**Scale/Scope**: Single-user personal app (MVP); no multi-tenancy, no real-time sync

## Key Implementation Decisions

### Authentication Flow

```
Sign Up / Login → Route Handler → bcrypt verify → JWT pair signed with HS256
→ access token (15 min) in HttpOnly cookie "at"
→ refresh token (7 days) in HttpOnly cookie "rt"
→ Next.js middleware reads "at" cookie → validates → allows or redirects
→ Client receives no token in JSON body
```

### Master Category Seeding Strategy

A dedicated table `system_categories` holds the global master data (inserted by
migration `002`). On every user `POST /api/v1/auth/register`, the API runs:

```sql
INSERT INTO categories (id, name, type, user_id, is_system_derived, created_at, updated_at)
SELECT gen_random_uuid(), name, type, $new_user_id, true, now(), now()
FROM system_categories;
```

This gives each user their own editable copy of the master categories while keeping
`system_categories` as the immutable reference.

### Amounts Storage

All amounts are stored as `INT` in **đồng** (VND). VND has no decimal subunits in
everyday use. Display uses the shared `currency.ts` formatter:
`1500000 → "1.500.000 ₫"`. No `float` or `decimal` is used anywhere.

### Markdown Safety

`markdown-it` parses the note string → raw HTML. `DOMPurify.sanitize()` strips all
event handlers and `<script>` tags. Result is set via React's `dangerouslySetInnerHTML`
exclusively in the `<MarkdownRenderer>` component.

## Migration Strategy

| Migration                               | Purpose                                                                                                                         |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `20260330000001_init_schema`            | Create `users`, `system_categories`, `categories`, `transactions` tables with all constraints, indexes, and soft-delete columns |
| `20260330000002_seed_system_categories` | INSERT 5 income + 10 expense master categories into `system_categories`                                                         |

New schema changes → `prisma migrate dev --name <description>` (dev).  
Production deploy → `prisma migrate deploy` (applies pending migrations only).  
Direct DDL edits to the database are **PROHIBITED** (Constitution Principle V).
